#### 1.1 容器監控基礎

**理論部分：**
```
容器 vs VM 監控的差異
├─ 資源隔離機制 (cgroups v1 vs v2)
├─ 容器快速生命週期對監控的影響
├─ Docker daemon 指標
└─ 容器日誌驅動程序整合

Docker 監控工具生態
├─ docker stats (原生)
├─ cAdvisor (Google 官方)
├─ Prometheus Docker socket exporter
└─ 性能對比分析
```

**實踐部分：**
```bash
# 1. 啟用 Docker Prometheus metrics (實驗功能)
# /etc/docker/daemon.json
{
  "metrics-addr": "127.0.0.1:9323",
  "experimental": true
}

# 2. 部署 cAdvisor
docker run \
  --detach \
  --volume=/:/rootfs:ro \
  --volume=/var/run:/var/run:ro \
  --volume=/sys:/sys:ro \
  --volume=/var/lib/docker/:/var/lib/docker:ro \
  --publish=8080:8080 \
  --name=cadvisor \
  gcr.io/cadvisor/cadvisor:latest

# 3. 驗證指標收集
curl http://localhost:8080/metrics | head -30
```

#### 1.2 Ansible playbook：Docker 監控部署

```yaml
# roles/docker_monitoring/tasks/main.yml

---
- name: "部署 Docker 容器監控"
  block:

    - name: "啟用 Docker Prometheus metrics"
      copy:
        dest: /etc/docker/daemon.json
        content: |
          {
            "metrics-addr": "127.0.0.1:9323",
            "experimental": true,
            "debug": false
          }
      notify: "重啟 Docker daemon"

    - name: "部署 cAdvisor (容器)"
      docker_container:
        name: cadvisor
        image: gcr.io/cadvisor/cadvisor:latest
        state: present
        restart_policy: unless-stopped
        ports:
          - "8080:8080"
        volumes:
          - /:/rootfs:ro
          - /var/run:/var/run:ro
          - /sys:/sys:ro
          - /var/lib/docker/:/var/lib/docker:ro
        command: "--port=8080"

    - name: "部署 Docker Socket exporter"
      docker_container:
        name: docker_exporter
        image: prometheusnet/docker_exporter:latest
        state: present
        restart_policy: unless-stopped
        ports:
          - "9417:9417"
        volumes:
          - /var/run/docker.sock:/var/run/docker.sock

    - name: "更新 Prometheus 配置 - 添加 Docker targets"
      blockinfile:
        path: /etc/prometheus/prometheus.yml
        insertafter: "# Docker Monitoring"
        block: |
          - job_name: 'docker-daemon'
            static_configs:
              - targets:
                  - 'localhost:9323'

          - job_name: 'cadvisor'
            static_configs:
              - targets:
                  - 'localhost:8080'

          - job_name: 'docker-exporter'
            static_configs:
              - targets:
                  - 'localhost:9417'
      notify: "重載 Prometheus"

  handlers:
    - name: "重啟 Docker daemon"
      systemd:
        name: docker
        state: restarted

    - name: "重載 Prometheus"
      uri:
        url: "http://localhost:9090/-/reload"
        method: POST
```

**重要指標：**

**告警規則示例：**

```yaml
- alert: ContainerHighCpuUsage
  expr: |
    (sum(rate(container_cpu_usage_seconds_total{name!=""}[5m])) 
    by (name) / count(count(container_cpu_usage_seconds_total{name!=""}) 
    by (cpu)) by (name)) * 100 > 80
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "容器 {{ $labels.name }} CPU 使用率過高"

- alert: ContainerMemoryUsageHigh
  expr: |
    (sum(container_memory_usage_bytes{name!=""}) by (name) 
    / sum(container_spec_memory_limit_bytes{name!=""}) by (name)) * 100 > 90
  for: 5m
  labels:
    severity: critical
  annotations:
    summary: "容器 {{ $labels.name }} 記憶體即將耗盡"
```

#### 1.5 故障排除與最佳實踐

**常見問題：**

```
Q1: cAdvisor 指標基數過高，影響 Prometheus 性能
A: 使用 relabel_configs 降低採樣率或篩選特定容器

Q2: Docker metrics endpoint 不穩定
A: 使用 cAdvisor 作為主要數據源，Docker metrics 作為補充

Q3: 容器頻繁重啟導致告警風暴
A: 增加 for 持續時間，使用 inhibit_rules 抑制相關告警
```

#### 4.2 應用埋點示例 (Go)

```go
// main.go - OpenTelemetry Instrumentation

package main

import (
	"context"
	"log"

	"go.opentelemetry.io/otel"
	"go.opentelemetry.io/otel/exporters/jaeger/rpc"
	"go.opentelemetry.io/otel/sdk/resource"
	tracesdk "go.opentelemetry.io/otel/sdk/trace"
	semconv "go.opentelemetry.io/otel/semconv/v1.17.0"
)

func initTracer() (*tracesdk.TracerProvider, error) {
	exporter, err := rpc.New(
		rpc.WithEndpoint("http://localhost:14250"),
	)
	if err != nil {
		return nil, err
	}

	tp := tracesdk.NewTracerProvider(
		tracesdk.WithBatcher(exporter),
		tracesdk.WithResource(resource.NewWithAttributes(
			context.Background(),
			semconv.ServiceNameKey.String("my-app"),
		)),
	)
	otel.SetTracerProvider(tp)

	return tp, nil
}

func main() {
	tp, err := initTracer()
	if err != nil {
		log.Fatal(err)
	}
	defer func() {
		if err := tp.Shutdown(context.Background()); err != nil {
			log.Fatal(err)
		}
	}()

	tracer := otel.Tracer("my-app")

	// 建立追蹤
	ctx, span := tracer.Start(context.Background(), "doWork")
	defer span.End()

	// 業務邏輯...
}
```

#### 5.1 PromQL 進階函數

**聚合函數高級用法：**

```promql
# 基於百分位的動態閾值
histogram_quantile(0.95, rate(request_duration_seconds_bucket[5m])) > 2

# 多維度聚合 (by vs without)
sum by (job, instance) (metric)
sum without (le) (metric)

# 子查詢
max_over_time(
  max(rate(container_cpu_usage_seconds_total[5m])) by (container)[5m:]
)[1h:5m]

# 二元操作符
# on / ignoring 修飾符用於標籤匹配
sum(increase(requests_total[5m])) by (job)
  / on() group_left()
sum(increase(requests_total[5m]) >= 1) by (job)
```

#### 5.2 異常檢測算法

**方法 1：Sigma-based 檢測**

```promql
# 當值超過 7 天平均 + 3σ 時告警
rate(http_requests_total[5m])
  >
(
  avg(rate(http_requests_total[5m]) offset 7d) 
  + 3 * stddev(rate(http_requests_total[5m]) offset 7d)
)
```

**方法 2：Predict Linear (預測算法)**

```promql
# 預測磁碟會在 4 小時內用完
predict_linear(
  node_filesystem_avail_bytes{device="/dev/sda1"}[1h],
  4*3600
) < 0
```

**方法 3：Rate of Change 檢測**

```promql
# 檢測錯誤率在上升
rate(errors_total[5m]) > 2 * rate(errors_total[1h] offset 1h)
```

#### 5.3 Python + Prophet 實現機器學習告警

```python
# anomaly_detector.py

from prometheus_client import CollectorRegistry, Gauge, start_http_server
from prophet import Prophet
import pandas as pd
import requests
import time

class AnomalyDetector:
    def __init__(self, prometheus_url):
        self.prom_url = prometheus_url
        self.registry = CollectorRegistry()
        self.anomaly_score = Gauge(
            'anomaly_score',
            'Anomaly detection score',
            ['metric'],
            registry=self.registry
        )

    def fetch_metrics(self, query, days=30):
        """從 Prometheus 拉取歷史數據"""
        end_time = int(time.time())
        start_time = end_time - (days * 86400)
        
        response = requests.get(
            f"{self.prom_url}/api/v1/query_range",
            params={
                'query': query,
                'start': start_time,
                'end': end_time,
                'step': 300  # 5 分鐘
            }
        )
        return response.json()['data']['result']

    def detect_anomaly(self, metric_name, query):
        """使用 Prophet 檢測異常"""
        data = self.fetch_metrics(query)
        
        if not data or not data[0]['values']:
            return None
        
        # 轉換為 DataFrame
        df = pd.DataFrame({
            'ds': pd.to_datetime([v[0] for v in data[0]['values']], unit='s'),
            'y': [float(v[1]) for v in data[0]['values']]
        })
        
        # 訓練 Prophet 模型
        model = Prophet(interval_width=0.95)
        model.fit(df)
        
        # 預測未來 1 小時
        future = model.make_future_dataframe(periods=12, freq='5min')
        forecast = model.predict(future)
        
        # 計算異常分數
        latest_value = df['y'].iloc[-1]
        upper_bound = forecast['yhat_upper'].iloc[-1]
        lower_bound = forecast['yhat_lower'].iloc[-1]
        
        if latest_value > upper_bound:
            anomaly_score = (latest_value - upper_bound) / (upper_bound - lower_bound)
        elif latest_value < lower_bound:
            anomaly_score = (lower_bound - latest_value) / (upper_bound - lower_bound)
        else:
            anomaly_score = 0
        
        self.anomaly_score.labels(metric=metric_name).set(anomaly_score)
        
        return anomaly_score

if __name__ == '__main__':
    detector = AnomalyDetector('http://localhost:9090')
    start_http_server(8001)
    
    while True:
        detector.detect_anomaly(
            'cpu_usage',
            'rate(node_cpu_seconds_total{mode="user"}[5m])'
        )
        time.sleep(300)  # 每 5 分鐘檢測一次
```
